Kapua Cucumber Reports

Functional tests

Functional tests are testing basic functionality of Kapua Services. Functionality describes public API of service, as an example CRUD (Create Read Update Delete) operations. Each service is tested in isolation, with all other services mocked and not included in test.

  • User service
  • Account service
  • Device registry service
  • Authorization service
  • Job service

Integration tests

Integration tests exercise multiple live Kapua Services. These are meant to test higher level functionality of Kapua. Example of such test would be Kapua user update functionality, where User service in combination with Authentication and Authorization service is tested. All services are real services and are not mocked.

  • User service
  • User lockout tests
  • Device service
  • Device lifecycle tests
  • Device registration with active MQTT broker
  • Device connection with config file broker ip
  • Device connection with system environment broker ip
  • Device connection with undefined broker ip
  • Access control for broker connections
  • Base device data publishing tests
  • Access control for device management
  • Tests for management of connection and user copuling
  • Datastore with transport client
  • Datastore with REST interface
  • Tagging service tests
