Kapua Cucumber Reports

Functional tests

Functional tests are testing basic functionality of Kapua Services. Functionality describes public API of service, as an example CRUD (Create Read Update Delete) operations. Each service is tested in isolation, with all other services mocked and not included in test.

  • Authorization service
  • User service
  • Account service
  • Device registry service

Integration tests

Integration tests are testing multiple live Kapua Service. This are meant to test higher level functionality of Kapua. Example of such test would be Kapua user update functionality, where User service in combination with Authentication and Authorization service is tested. All services are real services and are not mocked.

  • User service
  • Device registration with active MQTT broker
